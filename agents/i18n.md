# Playbook — Panduan Multibahasa (i18n)

Dokumen ini adalah aturan wajib mengenai penanganan bahasa di proyek ini.
Tujuannya adalah untuk menjaga konsistensi antara interaksi Agent, penulisan source code, dan tampilan UI aplikasi.

## 1. Bahasa Interaksi Agent & Dokumentasi (Bahasa Indonesia)
Semua instruksi, penjelasan, dan dokumentasi di dalam direktori `agents/` **wajib** menggunakan **Bahasa Indonesia** yang baku.
- Saat berinteraksi (chat) dengan developer, AI Agent **harus** merespons dalam Bahasa Indonesia.
- Jangan gunakan bahasa lain kecuali untuk menjelaskan istilah teknis (sebagai *loan words*).

## 2. Bahasa Source Code (Bahasa Inggris)
Semua entitas di dalam *codebase* (`src/`, file konfigurasi, dll.) **wajib** menggunakan **Bahasa Inggris**.
- **Penamaan Variabel & Fungsi**: Gunakan camelCase/PascalCase dalam Bahasa Inggris (contoh: `fetchUserData`, `SectionLayout`).
- **Komentar Kode (Code Comments)**: Gunakan Bahasa Inggris untuk konsistensi ekosistem globa.
- **Commit Messages**: Ikuti gaya *conventional commits* berbahasa Inggris (contoh: `feat: implement i18n context`, `fix: header styling`).

## 3. Bahasa Konten UI (Localization / i18n)
DILARANG KERAS melakukan *hardcode* pada *string* teks antarmuka (UI). Aplikasi ini menggunakan `LanguageContext` dan terjemahan dinamis melalui `src/lib/i18n.ts`.

> **[WAJIB] Aturan Komponen UI:**
> 1. Jangan masukkan teks mentah ke dalam JSX.
>    - ❌ Salah: `<button>Lanjut</button>`
>    - ❌ Salah: `<button>Next</button>`
> 2. Selalu impor dan gunakan dukungan i18n dari proyek ini.
>    - ✅ Benar: Gunakan fungsi terjemahan yang tersedia, seperti `t('button.next')`.
> 3. Jika membuat teks UI baru, **kamu (Agent) wajib memperbarui seluruh berkas bahasa** (contoh: `id.json` dan `en.json` di direktori _locales_ atau _dictionary_ terkait) secara serentak.
